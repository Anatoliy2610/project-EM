## Проект: Система управления бизнесом (Упрощённая версия)

Описание идеи:
Вы создаёте веб-приложение для управления командой внутри компании. Система позволит:
- Регистрироваться и управлять аккаунтами пользователей
- Создавать команды (компании) и управлять ими
- Назначать задачи сотрудникам и отслеживать их выполнение
- Оценивать работу сотрудников
- Создавать встречи и просматривать календарь событий

Цель — сделать одностраничное MVP-приложение без микросервисов, с понятной архитектурой и минимальным фронтендом.

---

## Выбор стека (по желанию)

---

### Вариант 1: Стек с Django

Технологии:
- Django 4.x
- Django REST Framework (DRF) — для API (если планируется frontend отдельно)
- PostgreSQL или SQLite
- Django Admin — для администрирования
- Bootstrap/Jinja (если нужен HTML-интерфейс)
- Celery (опционально, если хочется отложенные задачи)
- Simple JWT (если нужен токенный доступ)

Почему Django подойдёт:
- Отличный выбор для новичка
- Есть встроенная панель администратора
- Управление пользователями, моделями и правами — из коробки
- Хорошо подходит для Monolith-приложений

---

### Вариант 2: Стек с FastAPI

Технологии:
- FastAPI
- SQLAlchemy + Alembic (миграции)
- PostgreSQL или SQLite
- Pydantic (валидация)
- fastapi-users (готовое решение для авторизации)
- sqladmin или fastapi-admin — готовая админка
- Jinja2 (если нужен веб-интерфейс)
- Uvicorn — сервер запуска
- Docker (опционально)

Почему FastAPI подойдёт:
- Быстрый и современный фреймворк
- Подходит для API-first проектов
- Гибкий, удобный для изучения архитектуры
- Отлично комбинируется с frontend-фреймворками (Vue, React)

## Функциональность проекта (для обеих реализаций)

### 1. Пользователи
- Регистрация с email и паролем
- Авторизация / выход
- Роли: обычный пользователь, менеджер, админ команды
- Обновление профиля
- Удаление аккаунта (без восстановления)
- Привязка к команде по коду (опционально)

### 2. Команды
- Админ создаёт компанию / команду
- Добавление/удаление пользователей в команде
- Просмотр состава команды
- Назначение ролей (менеджер, сотрудник)

### 3. Задачи
- Создание задач руководителем
- Назначение исполнителя
- Описание, дедлайн, статус
- Изменение / удаление
- Комментарии (упрощённый чат внутри задачи)
- Статусы: открыто, в работе, выполнено

### 4. Оценка работы
- Руководитель оценивает выполненные задачи: баллы от 1 до 5
- Пользователь видит свои оценки
- Вывод средней оценки за период

### 5. Встречи
- Назначение встречи: дата, время, участники
- Проверка времени (чтобы не накладывались события)
- Список встреч пользователя
- Возможность отменить встречу

### 6. Календарь
- Простое отображение задач и встреч по дням
- Месячный и дневной вид в виде текстовой таблицы
- Автоматическое добавление задач и встреч

### 7. Админ-панель
- Готовая админка: управление пользователями, командами, задачами
- Django Admin для Django
- sqladmin/fastapi-admin для FastAPI



## Примерные шаги для реализации (любого стека)
1. Настройка проекта (FastAPI или Django)
2. Создание моделей: User, Team, Task, Meeting, Evaluation
3. Реализация авторизации (логин/регистрация)
4. Связь пользователей с командами
5. CRUD для задач, встреч, оценок
6. Календарь на основе дат (дневной и месячный список)
7. Админ-панель для управления
8. Тестирование

## Совет для студента
- Используй SQLite на первом этапе — быстрее настроить
- Начни с User + Task — это базовая логика
- Не спеши с фронтом — работай с Swagger или Django admin
- Делай коммиты часто и пиши краткие описания
